# Юзкейс: Асистент ріелтора

## Мета та короткий опис

**Мета:** допомогти ріелтору швидко знаходити опорні факти та контекст у власних документах — об’єктах, клієнтах, договорах і регуляторних нотатках — і відповідати клієнту або готувати матеріали **на основі джерел**, а не з пам’яті.

**Короткий опис:** асистент отримує запит природною мовою («Які квартири під оренду в районі Старе Місто», «Що ми домовились з Іваном про бюджет?», «Які угоди зроблені за останній квартал», «Який відсоток податку на купівлю у 2026 році», «Який перелік документів потрібен для зустрічі з кредитним радником»), знаходить релевантні фрагменти з корпусу документів і формує відповідь з опорою на знайдений контекст. Основа для подальших кроків: фільтрація за атрибутами, інтеграції, агентські сценарії — але це вже поза мінімальним RAG. У повній версії продукту сюди додаються CRM, права доступу, свіжі оголошення з майданчиків, моделі прогнозу тощо.

## Склад документів і даних

| Джерело | Зміст | Формат |
|--------|--------|--------|
| Каталог обєктів нерухомості | Продаж / оренда, ціна, площа, кімнати, район, статус, короткий опис | CSV + текстові поля |
| Розширений опис об’єкта | Маркетинговий текст, особливості | Markdown / текст |
| Технічний паспорт, план приміщень | Площі, перепланування, схема | PDF / зображення · **advanced** |
| Виписка / реєстраційні дані (умовно) | Обтяження, частки | PDF · **advanced** |

### Клієнти та угоди

| Джерело | Зміст | Формат |
|--------|--------|--------|
| Клієнти та звернення (рядок = один процес) | Один клієнт може мати кілька рядків: дата початку роботи, обраний `listing_id`, дата угоди, комісія; для активних звернень поля угоди можуть бути порожні; нотатки про домовленості | `clients.csv` |
| Договір співпраці з ріелтором | Комісія, термін, ексклюзивність | PDF · **advanced** (без OCR — як текст у воркшопі) |
| Договір купівлі / оренди | Умови, дати, застава | PDF · **advanced** |
| Платіжні документи за послуги | Сума, призначення, дата | PDF або рядки в CSV · **advanced** |

### Політики та ринок

| Джерело | Зміст | Формат |
|--------|--------|--------|
| Податки та обмеження в юрисдикції | ПДФО, податок / збір при купівлі, **ставки на конкретний рік (наприклад 2026)** з коротким поясненням | Кілька коротких Markdown-статей або один файл «Податки_2026» |
| Іпотека / робота з банком | **Чеклист документів для зустрічі з кредитним радником**, типові терміни етапів | Markdown |
| Внутрішні інструкції агентства | Скрипти, чеклисти показу | Markdown |

## Питання для простого кейсу (поточний воркшоп)

- Яка **ціна** у об’єкта за адресою …?
- Скільки **кімнат** і яка **площа** у квартири …?
- Чи є в описі згадка про **ремонт** / **паркінг** / **школу**?
- Чи дозволені **тварини**? Які обмеження?
- Що зазвичай відбувається **до** підписання попереднього договору?
- Які **податкові** аспекти для продажу в цьому регіоні?
- Яке **обмеження** для короткострокової оренди (якщо є в тексті)?
- Який **бюджет** у клієнта …?
- Який **район** шукає клієнт …?
- Які **квартири під оренду** в районі **Старе Місто**?
- **Що ми домовились** з Іваном **про бюджет**?
- Які **угоди зроблені за останній квартал**?
- Який **відсоток податку на купівлю у 2026 році**?
- Який **перелік документів** потрібен для зустрічі з **кредитним радником**?

## Обмеження поточного воркшопу

На цьому воркшопі ми свідомо беремо **частковий** кейс «асистент ріелтора»: невеликий **синтетичний** набір документів (переважно текст і спрощені таблиці), без повного CRM, без реальних PDF і без парсингу зображень планів.
**Головна мета воркшопу** — пройти **end-to-end**: завантажити документи → розбити на чанки → ембеддинги → індекс → retrieval → збір контексту → промпт → відповідь з опорою на джерела, плюс коротка **діагностика** поганих відповідей і міні-оцінка. Повний продукт для ріелтора ширший; тут ми формуємо **чітке розуміння** ланцюга RAG, а не всіх інтеграцій ринку.

## Next level: Розширені питання (складніші формати)

Більш просунутий RAG: **multi-RAG**, окремі індекси за типами документів, OCR, зображення, мапи, SQL/noSQL + семантика тощо.

- Який об’єкт має **найнижчу ціну за м²** серед активних об’єктів **у продажу**?
- Скільки **активних** клієнтів із бюджетом вище …?
- «Підбери 3 об’єкти під запит клієнта Z» — потрібні **фільтри** + семантика (гібрид або окремий інструмент).
- Витягнути **точну суму комісії** з договору лише з PDF — **advanced** (парсинг/OCR).
- «Скільки спалень на плані?» — **advanced** (мультимодальна модель аналізу зображень або ручний текстовий опис плану в корпусі).

## Next-next level: Можливі розширення (поза межами RAG)

Ідеї для продукту або наступних курсів; **не входять** у базовий RAG-воркшоп, але показують напрям розвитку.

- **Стягнення нових оголошень з OLX** — періодичний скрейп/API, дедуплікація, оновлення індексу; потребує стабільних контрактів з даними, обробки помилок і юридичних обмежень.
- **Модель прогнозу ціни за м²** залежно від **району** та **кількості спалень** — окремий ML/регресійний шар; RAG може пояснювати фактори з тексту, але сам прогноз — це не класичний retrieval.
- Календар показів.
- Підсумовування чатів у Telegram.
- Генерація e-mail із **підстановкою** імен та адрес на основі RAG.
